В ИДЗ не нужно создавать динамически обновляемые рисунки. Должно быть два статичных рисунка на разные типы входных данных.

Разработать калькулятор для вычисления характеристик равнобедренной трапеции, зная основания трапеции и высоту к основанию или основания трапеции и угол межу основанием и боковой стороной. Калькулятор должен вычислять:

1) диагонали;
2) неизвестные стороны;
3) периметр;
4) высоту от основания к боковой стороне.

Калькулятор должен включать:
1) группу переключателей (radio) для выбора типа входных данных (основания трапеции и высоту к основанию или основания трапеции и угол межу основанием и боковой стороной) и кнопку Показать;
2) картинку с геометрической фигурой и отмеченными на ней входными данными;
3) поля для ввода входных данных;
4) группу переключателей (checkbox) для выбора одной или нескольких вычисляемых характеристик;
5) кнопки Вычислить и Очистить.

В начальный момент времени на странице должна отобразиться картинка, соответствующей ей тип входных данных, отмеченный в группе переключателей, поля для ввода этих данных, а также вычисляемые характеристики, перечисленные в группе переключателей.

После выбора пользователем нового типа входных данных и клику на кнопку Показать, на странице при необходимости должны измениться подписи полей для ввода данных и рисунок.

После ввода пользователем входных данных, выбора вычисляемых значений и клику по кнопке Вычислить, на страницу вывести результаты расчетов.

При клике на кнопку Очистить – очистить поля для ввода.

На странице должны обрабатываться все возможные ошибки ввода данных. Использовать стили для обозначения ошибок ввода (не alert()).
